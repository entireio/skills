# Bring Your Entire Context to Your Agents with Skills

As agents become the primary interface for development, many developers are choosing to delegate command execution to their coding agents. The [Entire CLI](https://github.com/entireio/cli) already captures the context behind your code changes: prompts, transcripts, and the decisions that led to each change, alongside your git history. Today we're making that data easier for your agent to use.

## Make your agent more powerful

Our [skills](https://docs.entire.io/skills/overview) repository contains agent-invokable workflows that teach your agent how to use the Entire CLI. Instead of looking up syntax or running commands yourself, you can ask in plain language and let your agent drive the workflow.

\<**add screen recording of a skill in action\>**

This initial release enables your agent to:

* **Search past work**\- The [`search`](https://github.com/entireio/skills/blob/main/skills/search/SKILL.md) skill finds prior work in your Entire history by topic, repo, branch, author, or time window, so your agent can bring past context into the current task before making changes.  
* **Explain why code exists \-** The [`explain`](https://github.com/entireio/skills/blob/main/skills/explain/SKILL.md) skill traces a function, file, or line back to the session that created it, so your agent can answer the intent behind code, not just describe what the code currently does.  
* **Investigate changes \-** The [`what-happened`](https://github.com/entireio/skills/blob/main/skills/what-happened/SKILL.md) skill combines `git blame` with Entire checkpoint context to explain why a specific block of code looks the way it does. It is useful during code review or when you are investigating a regression.  
* **Hand off between agents \-** The [`session-handoff`](https://github.com/entireio/skills/blob/main/skills/session-handoff/SKILL.md) picks up where another agent left off by reading the saved or active session and summarizing the task state, important discoveries, blockers, and next steps.  
* **Turn sessions into skills \-** The [`session-to-skill`](https://github.com/entireio/skills/blob/main/skills/session-to-skill/SKILL.md) skill turns repeated agent sessions into reusable workflows via `SKILL.md` drafts. It extracts the process from real work instead of asking you to write idealized instructions from scratch.

## Cross-agent by design

All skills are cross-agent. Install them once and use the same workflows in [Agent Skills](https://agentskills.io)\-compatible tools, including Claude Code, Codex, Cursor, Copilot, Gemini CLI, OpenCode, Factory Droid, Kiro, and Pi.

## Try it

We published a [tutorial](https://docs.entire.io/skills/tutorial) for trying the skills from scratch. It walks through installation, and using the skills in real codebase.
